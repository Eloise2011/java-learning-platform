# Java Learning Platform — Project Guide

Single-page interactive Java curriculum with 22 topics, quizzes, exercises, resource tracking, and MySQL-backed persistence.

## Architecture

```
index.html              ← Self-contained SPA (deployable artifact)
├── CSS (inline <style>)    ← Light theme with WCAG AA contrast
├── HTML structure          ← Sidebar + main content area
└── JS (inline <script>)    ← All application logic
    ├── CURRICULUM data     ← 22 topics embedded as JSON
    ├── PHASES metadata     ← 6 phases (1 active, 5 planned)
    ├── State management    ← STATE object (currentView, completedTopics, activeTab)
    ├── Rendering           ← renderSidebar(), renderLesson(), renderPractice(), etc.
    ├── Persistence         ← API-first (MySQL via Flask), localStorage fallback
    └── Mermaid.js          ← CDN-loaded for site map diagram

src/
├── build.py               ← Master build script (assembles index.html)
├── curriculum/phase1.py   ← Phase 1 curriculum data (source of truth)
├── templates/head.html    ← HTML/CSS up to curriculum insertion point
├── templates/tail.html    ← JS app code + closing tags
└── app.js                 ← Extracted JS (reference only)

backend/
├── app.py                 ← Flask REST API (port 5001)
├── db.py                  ← MySQL connection helper
└── requirements.txt       ← flask, flask-cors, mysql-connector-python

dist/index.html            ← Build output (generated by src/build.py)
```

## Commands

```bash
# Build the HTML from curriculum data
python3 src/build.py

# Start the backend API (MySQL required)
cd backend && python3 app.py   # Runs on port 5001

# Test the API
curl http://localhost:5001/api/health
curl http://localhost:5001/api/progress

# Validate JS syntax
node --check dist/index.html  # Won't work (HTML); extract JS first
```

## Key Patterns

- **State:** `STATE` global object holds `currentView` ('welcome'|'topic'|'progress'|'sitemap'), `currentTopicId`, `completedTopics` (Set), `expandedPhases` (Set), `activeTab`
- **Rendering:** All render functions use `innerHTML`. Tab state preserved via `STATE.activeTab`. App-level views (Progress, SiteMap) controlled by `currentView`.
- **Persistence:** API-first to `http://localhost:5001/api/`, localStorage fallback. API calls wrapped in `apiFetch()` which sets `apiOnline` flag.
- **Quiz system:** Questions stored in `topic.quiz[]`. Answers persisted to both `localStorage` and MySQL `quiz_answers` table via `saveQuizAnswer()`. Previous answers restored on re-render via `getQuizAnswer()`.
- **Mermaid:** Lazy-rendered — diagram built in `buildMermaidDiagram()`, rendering deferred to `switchTab()` or `renderAppSiteMap()` when container is visible.
- **Curriculum data flow:** `src/curriculum/phase1.py` → `src/build.py` → `dist/index.html`

## Editing Lessons

Do NOT manually edit the CURRICULUM JSON in index.html. Instead:
1. Edit `src/curriculum/phase1.py` (the `TOPICS` list)
2. Run `python3 src/build.py` to regenerate `dist/index.html`
3. Copy `dist/index.html` to root or deploy directly

Each topic in the curriculum has: id, phase, phaseColor, phaseClass, title, hours, complexity, importance, textbook, summary, lesson (HTML string), quiz (array of {question, options, correct, explanation}), codingExercise ({instruction, hint, checkKeywords})

## Database

MySQL schema in `java_learning_platform` database:
- `users` — default user (id=1)
- `topic_progress` — completed topics
- `quiz_answers` — per-question answers with correctness
- `topic_resources` — extra-curriculum resources (books, links, podcasts, videos) with author field
- `topic_enhancements` — user feedback/review submissions

## Port Conflicts

- Port 5000: Used by macOS AirPlay Receiver. Backend uses port 5001 to avoid this.
- Kill stale backend processes: `lsof -i :5001` to find PID, then `kill -9 <PID>`
